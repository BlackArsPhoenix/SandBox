# TripoSR: изображение → 3D → STL на Windows 10

Готовый комплект для полностью локального восстановления 3D-модели по одному
изображению. После первоначального скачивания исходного кода и весов генерация
выполняется на компьютере пользователя без API-ключа.

> TripoSR реконструирует объект по одному видимому ракурсу. Невидимые поверхности
> модель предполагает, поэтому результат не является точным 3D-сканированием.

## Что находится в каталоге

| Файл | Назначение |
| --- | --- |
| `setup.ps1` | Проверяет систему, скачивает официальный TripoSR и создаёт окружение |
| `generate.ps1` | Генерирует OBJ/GLB и при необходимости сразу экспортирует STL |
| `start-web-interface.ps1` | Запускает локальный браузерный интерфейс |
| `convert-to-stl.ps1` | Масштабирует OBJ/GLB, проверяет сетку и создаёт STL |
| `scripts/convert_mesh_to_stl.py` | Реализация проверки и преобразования сетки |

Официальный репозиторий, Python-окружение, веса и результаты намеренно не
хранятся в Git. Они автоматически размещаются в игнорируемых каталогах
`vendor/`, `.venv/`, `.cache/` и `outputs/`.

## 1. Требования

Рекомендуемая конфигурация:

- Windows 10 x64 версии 22H2;
- видеокарта NVIDIA с поддержкой CUDA и минимум 6 ГБ видеопамяти;
- 16 ГБ оперативной памяти;
- не менее 15 ГБ свободного места;
- актуальный драйвер NVIDIA;
- Python 3.11 x64;
- Git for Windows;
- Microsoft Visual C++ Build Tools 2022.

### Установка обязательных программ

#### Драйвер NVIDIA

Скачайте актуальный драйвер:
<https://www.nvidia.com/Download/index.aspx>

После установки перезагрузите компьютер и выполните в PowerShell:

```powershell
nvidia-smi
```

Команда показывает название видеокарты, версию драйвера и доступную видеопамять.
Если программа не найдена или возвращает ошибку, продолжать установку TripoSR
пока рано.

#### Git и Python

Можно установить программы через `winget`:

```powershell
winget install --id Git.Git -e
winget install --id Python.Python.3.11 -e
```

Первая команда устанавливает Git, вторая — 64-битный Python 3.11. Если `winget`
недоступен, используйте официальные установщики:

- <https://git-scm.com/download/win>
- <https://www.python.org/downloads/windows/>

При ручной установке Python включите пункт `Install launcher for all users`.
Добавление Python в `PATH` необязательно: комплект использует стандартную
программу `py.exe`.

Проверьте установку в новом окне PowerShell:

```powershell
git --version
py -3.11 --version
```

#### Microsoft Visual C++ Build Tools

Некоторые зависимости содержат нативный код. Установите:
<https://visualstudio.microsoft.com/visual-cpp-build-tools/>

В установщике выберите нагрузку `Desktop development with C++` и компоненты:

- MSVC v143;
- Windows 10 или Windows 11 SDK;
- C++ CMake tools for Windows.

Отдельный CUDA Toolkit сначала не требуется: PyTorch содержит собственные
CUDA-библиотеки. Он понадобится только при ошибке сборки CUDA-расширения; решение
описано в разделе диагностики.

## 2. Получение репозитория

Если репозиторий ещё не скачан, команда ниже создаёт локальную копию:

```powershell
git clone https://github.com/BlackArsPhoenix/SandBox.git C:\AI\SandBox
```

Перейдите в каталог комплекта:

```powershell
cd C:\AI\SandBox\tools\image-to-3d\triposr-windows
```

Рекомендуется использовать короткий путь латиницей. Это уменьшает вероятность
ошибок старых нативных зависимостей.

## 3. Разрешение локальных PowerShell-скриптов

Следующая настройка разрешает текущему пользователю запускать локальные
подписанные и созданные им сценарии:

```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```

Если менять политику нельзя, каждый сценарий можно запускать однократно так:

```powershell
powershell -ExecutionPolicy Bypass -File .\setup.ps1
```

## 4. Автоматическая установка TripoSR

Для современной версии драйвера NVIDIA запустите:

```powershell
.\setup.ps1 -TorchChannel cu128
```

Сценарий последовательно:

1. проверит Windows, Git, Python и драйвер NVIDIA;
2. выведет сведения о видеокарте;
3. скачает официальный репозиторий TripoSR;
4. создаст изолированное окружение `.venv`;
5. установит актуальную CUDA-сборку PyTorch;
6. установит официальные зависимости и инструменты экспорта STL;
7. проверит совместимость пакетов и доступность CUDA.

Если канал `cu128` не устанавливается, сначала обновите драйвер. Для системы,
которая по техническим причинам должна использовать CUDA 12.6, доступен вариант:

```powershell
.\setup.ps1 -TorchChannel cu126
```

Процессорный режим предназначен только для проверки установки и будет очень
медленным:

```powershell
.\setup.ps1 -TorchChannel cpu
```

Повторный запуск безопасен: существующие веса и окружение сохраняются.

## 5. Подготовка изображения

Для более предсказуемой реконструкции:

- используйте одно изображение в PNG, JPEG или WEBP;
- объект должен полностью помещаться в кадр;
- фон должен быть однотонным и контрастным;
- части объекта не должны обрезаться краями;
- избегайте сильного размытия, глубоких теней и перекрывающих предметов;
- для печати полезнее чёткий силуэт, чем сложный фон.

Удалять фон вручную необязательно: TripoSR делает это автоматически.

## 6. Генерация модели из PowerShell

### Быстрый OBJ

```powershell
.\generate.ps1 `
  -ImagePath "C:\AI\references\object.jpg" `
  -OutputDirectory ".\outputs\object" `
  -ModelSaveFormat obj
```

Команда удаляет фон, реконструирует геометрию и сохраняет
`outputs\object\0\mesh.obj`.

Первый запуск скачивает открытые веса `stabilityai/TripoSR` в `.cache`. Это
несколько гигабайт и может занять продолжительное время. API-ключ не нужен.

### Сразу подготовить STL заданного размера

```powershell
.\generate.ps1 `
  -ImagePath "C:\AI\references\object.jpg" `
  -OutputDirectory ".\outputs\object-print" `
  -ModelSaveFormat obj `
  -ExportStl `
  -TargetHeightMillimeters 64 `
  -VoxelPitchMillimeters 0.35
```

Дополнительные параметры означают:

- `TargetHeightMillimeters 64` — итоговая высота 64 мм;
- `VoxelPitchMillimeters 0.35` — перестроение поверхности с шагом 0,35 мм для
  получения замкнутой сетки;
- результат — `outputs\object-print\0\mesh.stl`.

Рекомендуемые значения вокселя после задания физического размера:

| Шаг | Эффект |
| --- | --- |
| `0.25` мм | Больше деталей, больше полигонов и расход памяти |
| `0.35` мм | Сбалансированный вариант для миниатюры |
| `0.50` мм | Более прочная, но сглаженная поверхность |
| `0` | Не перестраивать сетку |

### Повысить детализацию исходной поверхности

```powershell
.\generate.ps1 `
  -ImagePath "C:\AI\references\object.png" `
  -OutputDirectory ".\outputs\object-detailed" `
  -MarchingCubesResolution 384
```

Значение `384` создаёт больше полигонов и требует больше видеопамяти. При ошибке
нехватки памяти используйте стандартное `256` или минимальное `128`.

### Создать текстурированную модель

```powershell
.\generate.ps1 `
  -ImagePath "C:\AI\references\object.png" `
  -OutputDirectory ".\outputs\object-textured" `
  -ModelSaveFormat glb `
  -BakeTexture `
  -TextureResolution 2048
```

GLB подходит для Blender, Godot и просмотра. STL не хранит цвет и текстуры,
поэтому для фотополимерной печати этот этап не нужен.

## 7. Локальный веб-интерфейс

Запустите:

```powershell
.\start-web-interface.ps1
```

Не закрывая окно PowerShell, откройте:
<http://127.0.0.1:7860>

Интерфейс работает локально. Изображение не отправляется внешнему сервису;
интернет используется только для первоначального скачивания кода и весов.
Остановка выполняется сочетанием `Ctrl+C` в окне PowerShell.

## 8. Отдельное преобразование OBJ/GLB в STL

Если модель уже сгенерирована или отредактирована в Blender:

```powershell
.\convert-to-stl.ps1 `
  -InputMeshPath ".\outputs\object\0\mesh.obj" `
  -OutputStlPath ".\outputs\object\0\mesh.stl" `
  -TargetHeightMillimeters 64 `
  -VoxelPitchMillimeters 0.35
```

Конвертер:

- объединяет геометрию сцены;
- применяет трансформации;
- удаляет вырожденные элементы;
- исправляет направление нормалей;
- масштабирует модель;
- помещает её на плоскость `Z=0`;
- при заданном шаге создаёт заполненную замкнутую поверхность;
- сообщает габариты, число граней, замкнутость и число компонентов.

Воксельное перестроение сглаживает детали. Сначала попробуйте конвертацию без
него, а затем включайте, если проверка сообщает `Watertight: False`.

## 9. Проверка перед печатью

Откройте STL в одном из приложений:

- Microsoft 3D Builder;
- Blender с дополнением `3D Print Toolbox`;
- PrusaSlicer;
- Lychee Slicer;
- CHITUBOX.

Проверьте:

1. физический размер в миллиметрах;
2. отсутствие отдельных плавающих компонентов;
3. минимальную толщину тонких деталей;
4. наличие опор под нависающими элементами;
5. отсутствие закрытых внутренних полостей с жидкой смолой.

Автоматическая замкнутость не гарантирует механическую прочность. Тонкие детали
может потребоваться утолстить вручную в Blender.

## 10. Диагностика

### Запуск сценариев запрещён

Сообщение содержит `running scripts is disabled`.

```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```

Команда разрешает текущему пользователю запуск локальных сценариев.

### Не найден Python 3.11

Проверьте доступные версии:

```powershell
py --list
```

Если установлен Python 3.10, используйте:

```powershell
.\setup.ps1 -PythonVersion 3.10 -TorchChannel cu128
```

### `CUDA доступна: False`

Проверьте драйвер:

```powershell
nvidia-smi
```

Затем повторите установку нужного канала:

```powershell
.\setup.ps1 -TorchChannel cu128
```

Не устанавливайте CPU-сборку в то же окружение после CUDA-сборки.

### Ошибка компилятора Microsoft C++

Откройте Visual Studio Installer и добавьте `Desktop development with C++`,
MSVC v143 и Windows SDK. После установки откройте новое окно PowerShell и
повторите `setup.ps1`.

### `CUDA_HOME is not set` или не найден `nvcc`

Нативной зависимости потребовался полный CUDA Toolkit. Узнайте канал PyTorch,
использованный при установке, и установите соответствующую основную версию:

- `cu128` → CUDA Toolkit 12.8;
- `cu126` → CUDA Toolkit 12.6.

Загрузка: <https://developer.nvidia.com/cuda-downloads>

После установки перезагрузите компьютер и проверьте:

```powershell
nvcc --version
```

### Нехватка видеопамяти

Закройте игры, Blender и другие GPU-программы. Затем снизьте разрешение:

```powershell
.\generate.ps1 `
  -ImagePath "C:\AI\references\object.png" `
  -MarchingCubesResolution 128
```

Стандартная конфигурация TripoSR использует около 6 ГБ видеопамяти.

### Долгое ожидание на первом запуске

Проверьте содержимое `.cache\huggingface`. При первом запуске скачиваются веса;
последующие генерации используют локальную копию.

### STL остаётся незамкнутым

Повторите конвертацию с шагом вокселя:

```powershell
.\convert-to-stl.ps1 `
  -InputMeshPath ".\outputs\object\0\mesh.obj" `
  -OutputStlPath ".\outputs\object\0\mesh-watertight.stl" `
  -TargetHeightMillimeters 64 `
  -VoxelPitchMillimeters 0.35
```

## 11. Обновление

Обновить официальный исходный код и проверить зависимости:

```powershell
.\setup.ps1 -TorchChannel cu128 -UpdateSource
```

Перед обновлением завершите веб-интерфейс и генерацию.

## 12. Удаление локальных данных

Чтобы освободить место, можно удалить внутри этого каталога:

- `.venv` — Python-окружение;
- `vendor` — официальный исходный код;
- `.cache` — веса и кэш пакетов;
- `outputs` — сгенерированные модели.

Сценарии и инструкция останутся в Git. После удаления для повторной установки
достаточно снова выполнить `setup.ps1`.
